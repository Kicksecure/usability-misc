# TODO: dist-installer-cli standalone fallbacks

## Background

`log()` in `helper-scripts/usr/libexec/helper-scripts/log_run_die.sh` now calls
both `stecho` and `sanitize-string` (the latter via `has sanitize-string`).
These are external commands/functions that are available in a full Whonix install
but are NOT available when `dist-installer-cli` runs in standalone mode (e.g.
on a plain Debian system before anything is installed).

## Required patch

In `usr/bin/dist-installer-cli`, after the line:
```
source /usr/libexec/helper-scripts/log_run_die.sh
```
(currently around line 389), add the following fallback definitions:

```bash
if ! has stecho; then
  stecho() {
    printf '%s\n' "$*"
  }
fi
if ! has sanitize-string; then
  sanitize-string() {
    cat
  }
fi
```

`has` is already sourced earlier (around line 238) so it is available at this point.

The `sanitize-string` fallback simply passes stdin through unchanged (`cat`).
This is acceptable for standalone mode: the CLI runs in a controlled terminal
context where the log output length limits that `sanitize-string` enforces are
not needed.

## Why this file was not patched automatically

`usr/bin/dist-installer-cli` is ~188 KB. The GitHub MCP write APIs require the
full file content in a single call body and that size exceeds the limit that
works reliably (~85 KB observed). Apply the patch manually.

The standalone variant (`usr/share/usability-misc/dist-installer-cli-standalone`)
is auto-generated by `usr/share/usability-misc/build-dist-installer-cli` and
must NOT be edited directly; re-run that script after patching the source.
